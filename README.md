# local-dependency-resolver

A Claude Code plugin for **Maven** projects. Lets Claude read and edit the
source of locally checked-out Maven dependencies directly — instead of
inspecting JARs.

## What it does

When you ask Claude to make a change in a Java project (e.g.
`~/projects/project-a/`) that involves a Maven dependency on another project
of yours (e.g. `com.example:project-b`), the bundled
`resolve-local-maven-dependency` skill lets Claude:

1. Look up the dependency's `groupId:artifactId` in a generated table.
2. Find the corresponding folder under `~/projects/` (e.g.
   `~/projects/project-b/`).
3. Read and edit the dependency's source files directly.

The table is auto-generated by scanning every top-level `pom.xml` under
`~/projects/` and reading each folder's `git remote.origin.url`.

## Requirements

- **Claude Code** — recent enough to support the `/plugin` command.
- **Python 3** on `PATH` — used to (re)generate the lookup table.
- **Git** on `PATH` — used to read each project's remote URL.
- All Java/Maven project checkouts placed as immediate subdirectories of a
  single **projects root** (each with a top-level `pom.xml`). The default
  root is `~/projects`; see [Configuration](#configuration) to point it
  elsewhere.

## Installation

In Claude Code, add the marketplace and install the plugin:

```
/plugin marketplace add git@github.com:LarsEliasNielsen/local-dependency-resolver.git
/plugin install local-dependency-resolver@local-dependency-resolver
```

The first `<…>` is `local-dependency-resolver` (the marketplace `name`); the
second is the plugin name within it — they happen to be identical.

To update later:

```
/plugin marketplace update
```

## Configuration

### Projects root

The skill needs to know where your Java projects live. Set
`LOCAL_DEPENDENCY_RESOLVER_ROOT` to that folder **before launching Claude
Code**, then restart Claude Code for it to take effect.

PowerShell (persistent, current user):

```powershell
[Environment]::SetEnvironmentVariable("LOCAL_DEPENDENCY_RESOLVER_ROOT", "D:\dev", "User")
```

Bash / WSL / macOS / Linux (add to your shell rc):

```bash
export LOCAL_DEPENDENCY_RESOLVER_ROOT="$HOME/code"
```

If you skip this step, the skill defaults to `~/projects`
(`%USERPROFILE%\projects` on Windows). That default is fine if your projects
are already there.

The lookup table records the root it was generated from in its header line,
so the skill always resolves paths against the value it was generated with.

You can also override per-run when invoking the generator script directly:

```
python "<path-to-script>" --root D:\dev
```

### Python interpreter

The skill calls `python` from `PATH`. If your system uses a different name
(`py` on Windows, `python3` on some Linux distributions), either:

- create an alias / shim so `python` resolves to your interpreter, or
- edit the SKILL.md inside your installed plugin to use the alternative
  command.

### Staleness threshold

The skill auto-regenerates the lookup table when it's older than **14 days**.
To change that threshold, edit the value in the installed `SKILL.md`
(under "Check freshness first").

### Output file location

By default, `local-dependencies.md` is written next to the generator script.
Override per-run with `--output <path>` if you want it elsewhere.

## Usage

Once your projects root is configured (see above), the skill triggers
automatically. When you ask Claude something that involves a Maven dependency
whose source is also checked out locally, Claude will resolve it and
read/edit files in the corresponding sibling folder.

On the first invocation after install, the lookup table
(`local-dependencies.md`) does not exist yet, so the skill generates it by
running its bundled Python script. This usually takes a couple of seconds for
~50 projects.

After that, the skill regenerates the table only when:

- the table is more than **14 days** old,
- the file is missing,
- you mention a project that has been added or removed since the table was
  last generated.

## Manually refreshing the table

You can also run the generator script directly. The script is bundled inside
the installed plugin; on Windows, the path will look something like:

```
%USERPROFILE%\.claude\plugins\<...>\local-dependency-resolver\skills\resolve-local-maven-dependency\generate-local-dependency-resolver.py
```

Run it with:

```
python "<path-to-script>"
```

It rewrites `local-dependencies.md` next to itself. Pass `--root <other-dir>`
or `--output <other-file>` to override the defaults.
