# local-dependency-resolver

A Claude Code plugin for **Maven** projects. Lets Claude read and edit the
source of locally checked-out Maven dependencies directly — instead of
inspecting JARs.

## What it does

When you ask Claude to make a change in a Java project that involves a Maven
dependency on another project of yours (e.g. `com.example:project-b`), the
bundled `resolve-local-maven-dependency` skill lets Claude:

1. Look up the dependency's `groupId:artifactId` in a generated table.
2. Find the corresponding folder under one of your configured project roots.
3. Read and edit the dependency's source files directly.

The table is auto-generated by scanning every top-level `pom.xml` across your
configured roots and reading each folder's `git remote.origin.url`.

## Requirements

- **Claude Code** — recent enough to support the `/plugin` command.
- **Python 3** on `PATH` — used to (re)generate the lookup table.
- **Git** on `PATH` — used to read each project's remote URL.
- All Java/Maven project checkouts placed as immediate subdirectories of one
  or more **project roots** (each with a top-level `pom.xml`). The defaults
  are `~/projects` and `~/Documents/Projects`; see
  [Configuration](#configuration) to add or change roots.

## Installation

In Claude Code, add the marketplace and install the plugin:

```
/plugin marketplace add https://github.com/LarsEliasNielsen/local-dependency-resolver.git
/plugin install local-dependency-resolver@local-dependency-resolver
```

To update later:

```
/plugin marketplace update
```

## Configuration

### Project roots

Open `paths.config.json`, which lives next to the generator script inside the
installed plugin:

```
# Windows
%USERPROFILE%\.claude\plugins\marketplaces\local-dependency-resolver\plugins\local-dependency-resolver\skills\resolve-local-maven-dependency\paths.config.json

# Unix / macOS
~/.claude/plugins/marketplaces/local-dependency-resolver/plugins/local-dependency-resolver/skills/resolve-local-maven-dependency/paths.config.json
```

It contains a JSON array of paths to scan:

```json
{
  "roots": [
    "~/projects",
    "~/Documents/Projects"
  ]
}
```

Add, remove, or reorder entries as needed. Each path supports `~` (home
directory) and environment variables (`%USERPROFILE%` on Windows, `$HOME` on
Unix). The defaults cover the most common locations on both Windows and
Unix/Mac. **Missing paths are silently skipped**, so it's safe to include
paths that may not exist on every machine.

After editing, regenerate the lookup table for the changes to take effect:

```
python "<path-to-script>"
```

Or just wait — the skill regenerates automatically when the table is more than
14 days old.

To override all configured roots for a single run:

```
python "<path-to-script>" --root D:\dev
```

### Python interpreter

The skill calls `python` from `PATH`. If your system uses a different name
(`py` on Windows, `python3` on some Linux distributions), either:

- create an alias / shim so `python` resolves to your interpreter, or
- edit the SKILL.md inside your installed plugin to use the alternative
  command.

### Staleness threshold

The skill auto-regenerates the lookup table when it's older than **14 days**.
To change that threshold, edit the value in the installed `SKILL.md`
(under "Check freshness first").

### Output file location

By default, `local-dependencies.md` is written next to the generator script.
Override per-run with `--output <path>` if you want it elsewhere.

## Usage

Once your project roots are configured (see above), the skill triggers
automatically. When you ask Claude something that involves a Maven dependency
whose source is also checked out locally, Claude will resolve it and
read/edit files in the corresponding local folder.

On the first invocation after install, the lookup table
(`local-dependencies.md`) does not exist yet, so the skill generates it by
running its bundled Python script. This usually takes a couple of seconds for
~50 projects.

After that, the skill regenerates the table only when:

- the table is more than **14 days** old,
- the file is missing,
- you mention a project that has been added or removed since the table was
  last generated.

## Manually refreshing the table

You can also run the generator script directly. The script is bundled inside the installed plugin:

```
# Windows
%USERPROFILE%\.claude\plugins\marketplaces\local-dependency-resolver\plugins\local-dependency-resolver\skills\resolve-local-maven-dependency\generate-local-dependency-resolver.py

# Unix / macOS
~/.claude/plugins/marketplaces/local-dependency-resolver/plugins/local-dependency-resolver/skills/resolve-local-maven-dependency/generate-local-dependency-resolver.py
```

Run it with:

```
python "<path-to-script>"
```

It rewrites `local-dependencies.md` next to itself, scanning whichever roots
are listed in `paths.config.json`. Pass `--root <other-dir>` to override all
configured roots for a single run, or `--output <other-file>` to write the
table elsewhere.

## Repository layout

```
.
├── .claude-plugin/
│   └── marketplace.json                       # marketplace manifest
├── plugins/
│   └── local-dependency-resolver/
│       ├── .claude-plugin/
│       │   └── plugin.json                    # plugin manifest
│       └── skills/
│           └── resolve-local-maven-dependency/
│               ├── SKILL.md                   # skill description + instructions
│               ├── generate-local-dependency-resolver.py
│               └── paths.config.json          # user-editable list of project roots
└── .gitignore
```

The generated `local-dependencies.md` is intentionally **not** committed —
it's user-specific and regenerated per machine on first use.
